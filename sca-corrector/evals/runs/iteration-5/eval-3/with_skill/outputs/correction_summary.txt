CÓDIGO PARTE B (ParteB.java)
- Intenta implementar un algoritmo greedy
- Itera dos veces sobre usuarios:
  1ª pasada: cuenta módulos por usuario
  2ª pasada: elimina usuarios si AMBOS módulos tienen cobertura >1
- ERROR CRÍTICO: Lógica incompleta

  * Resultado: puede devolver más usuarios de lo necesario O fallar en cobertura total
  * Algoritmo correcto requeriría: Set Cover, Greedy modificado, o búsqueda exhaustiva
